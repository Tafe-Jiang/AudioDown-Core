# AudioDown MCP UI Redesign Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Redraw the complete AudioDown Vue interface as a compact, responsive operations workspace using MCP-selected shadcn interaction patterns, while preserving the empty-core product boundary and the phase-two plugin installation workflows.

**Architecture:** The shadcn MCP registry is the design and interaction reference for shell, navigation, tables, forms, empty states, dialogs, loading states, and feedback. Because the MCP registry emits React/TSX examples, implementation uses the matching `shadcn-vue`/Reka UI components and `@lucide/vue`; React, Next.js, Radix React, and copied TSX are forbidden. Shared Vue primitives own visual consistency, views own page composition, and the existing typed API client remains the only network boundary.

**Tech Stack:** Vue 3.5, Vue Router 5, TypeScript 5.9, Vite 8, shadcn-vue 2.7.4, Reka UI 2.10.1, Tailwind CSS 4.3.2, `@lucide/vue` 1.24.0, Vitest 4, Vue Test Utils 2, Playwright 1.61.1, axe-core Playwright 4.12.1.

## Product and Visual Decisions

- The application is an operational tool, not a landing page.
- Keep Chinese interface labels and concise operational copy.
- Use one light theme only in this phase; do not add a theme switcher.
- Use a cool neutral canvas, white work surfaces, graphite navigation, coral
  primary actions, green success, amber warning, and red destructive states.
- Do not use gradients, decorative orbs, glass effects, oversized hero text,
  nested cards, or platform imagery.
- Keep card radius at 6px and control radius at 5px.
- Use `@lucide/vue` icons for navigation and commands. Icon-only controls need
  accessible names and tooltips.
- Desktop sidebar is 232px expanded and 64px collapsed. Mobile navigation is
  an off-canvas sheet, never a five-column text bar.
- Desktop content width is fluid up to 1440px. Data-heavy pages use available
  width instead of the current 1080px cap.
- Header height is 52px. Page titles inside the work area stay at 20-24px.
- Preserve all existing backend routes and response contracts. Only phase-two
  Task 15 API methods may be added.
- Search and Discover remain genuine empty states until content capability
  work is implemented. Do not add sample albums, tracks, platforms, or charts.
- Do not add credentials, Cookie UI, downloads, tasks, automatic updates,
  real platform labels, real repository URLs, or plugin-supplied HTML.
